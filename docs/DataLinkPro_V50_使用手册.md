## 第一章 快速入门

### 1.1 这个工具解决什么问题

如果你的日常工作中包含以下任何一种情况，这个工具就是为你准备的：

- 多个 Excel 表之间需要按某个字段匹配取数（类似 VLOOKUP）
- 每天/每周都在重复相同的报表处理流程
- 数据量大到 Excel 卡死/未响应
- 需要把多个格式相同的文件合并成一张总表
- 每次做完都想"这个流程能不能自动化"

DataLink Pro 的核心价值：**配一次规则，以后换数据就能一键重跑。不需要写代码，不需要学编程。**

### 1.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11（64位推荐） |
| 内存 | 4GB 以上（百万级数据建议 8GB+） |
| 磁盘空间 | 程序本身约 80MB，缓存另计 |
| 其他依赖 | 无。双击 exe 直接使用 |

### 1.3 安装与启动

本工具无需安装，下载后双击 `DataLinkPro.exe` 即可启动。

> **注意事项：**
>
> - 首次启动可能触发 Windows 安全警告，点击"更多信息"→"仍要运行"即可。
> - 程序会在同目录生成配置文件和缓存文件夹，请勿手动删除。
> - 建议把 exe 放在固定目录（如 `D:\Tools\`），不要放在桌面或下载文件夹。
> - 确保 `assets\icon.ico` 与 exe 在同一层目录下（用于显示程序图标）。

### 1.4 界面总览

程序界面分为三个核心区域：

| 区域 | 位置 | 功能 |
|------|------|------|
| Step 1：数据池配置 | 左侧 | 选择主表、添加匹配规则、加载数据池 |
| Step 2：输出任务设置 | 右侧 | 配置输出路径、新建/编辑输出任务 |
| Step 3：方案管理 | 底部 | 保存/加载/导出/导入预设方案、运行日志、进度条 |

底部的"启动任务"按钮是最终执行入口，旁边的"停止"按钮可在运行中安全中断任务。

## 第二章 操作流程（Step 1-2-3）

### 2.1 Step 1：配置数据池

#### 2.1.1 选择主表

主表是你的核心数据源，所有匹配结果都会合并到主表上。支持三种模式：

| 模式 | 适用场景 | 操作方式 |
|------|----------|----------|
| 单文件模式 | 主表是一个 Excel/CSV/Parquet 文件 | 点击"文件"按钮选择文件 |
| 多 Sheet 合并 | 主表数据分布在同一文件的多个工作表 | 点击"Sheet"按钮，Ctrl 多选 |
| 文件夹模式 | 多个结构相同的文件需要合并 | 点击"文件夹"按钮选择目录 |

**主表操作按钮说明：**

| 按钮    | 功能                            |
|-------|-------------------------------|
| 文件    | 选择单个数据文件（xlsx/xls/csv/parquet） |
| Sheet | 选择主表的工作表（支持多选合并）              |
| 列名    | 预览主表的列名列表（支持搜索、复制）            |
| 文件夹   | 选择文件夹作为主表（批量合并模式）             |
| 结构    | 打开主表结构配置器（选择保留哪些列）            |
| 多缓存   | 将文件夹内 Excel/CSV 批量转为 Parquet 格式 |
| 转缓存   | 将单个 Excel/CSV 转为 Parquet 格式   |

#### 2.1.2 添加匹配规则

匹配规则告诉程序："用主表的哪列去匹配目标表的哪列，然后把目标表的哪些列取回来"。

点击左侧"+ 添加"按钮，弹出规则配置窗口：

| 字段 | 说明 | 示例 |
|------|------|------|
| 目标文件 | 你要从中取数的文件 | `D:\data\价格表.xlsx` |
| Sheet | 目标文件的哪个工作表 | `Sheet1` 或点击"选择Sheet" |
| 列名匹配 | 勾选=用列名匹配，不勾选=用列字母（A/B/C） | 默认勾选 |
| 主表关联列 | 主表中用于匹配的列名 | `物料编码` |
| 目标关联列 | 目标表中对应的列名 | `物料号` |
| 目标取回列 | 要从目标表拿回来的列 | `单价,库存,供应商` |
| 列名前缀 | 防止列名重复的前缀 | `价格表_` |

> **操作技巧：**
>
> - 点击"目标列名"或"主表列名"按钮可打开列名预览窗口。
> - 预览窗口支持**搜索**和**Ctrl+C 复制**，可以直接粘贴到配置输入框中。
> - 所有预览窗口可以**自由切换层级**，点谁谁在最上面，不会互相锁定。
> - 列名匹配支持混写：`B:AA, 备注, F` 都能正确解析。
> - 支持多列联合关联：主表关联列填 `区域,门店号`，目标关联列也填对应的两列。

#### 2.1.3 加载数据池

主表和规则配好之后，点击"加载数据池"按钮。程序会分析主表列 + 规则取回列，生成完整的数据池列结构。

这一步成功后，右侧的输出任务才能正常配置（因为任务需要知道有哪些列可用）。

> 如果加载失败，错误信息会同时显示在弹窗和日志区，方便排查。

---

### 2.2 Step 2：设置输出任务

每个输出任务对应结果 Excel 中的一个 Sheet。点击右侧"+ 新增"按钮打开任务编辑窗口。

#### 2.2.1 基础配置

| 字段 | 说明 |
|------|------|
| Sheet 名称 | 结果文件中这个任务对应的工作表名称 |
| 筛选条件 | 可选。手动填写或点击"构建"按钮打开可视化筛选构建器 |
| 计算字段 | 可选。新增或修改列，详见 3.5 节 |
| 排序规则 | 可选。点击"设置排序"或右键列快速排序 |

#### 2.2.2 模式A：明细表

适用于"选择哪些列、什么顺序、是否重命名"的场景。

- **左侧**：可用数据源列表（支持搜索过滤）
  - 黑色 = 主表原有列
  - 蓝色 = 计算字段
- **右侧**：已选输出列（支持拖拽排序）

| 操作        | 方法                       |
|-----------|--------------------------|
| 添加列       | 左侧双击，或选中后点"添加 >"         |
| 全部添加      | 点"全选 >>"                 |
| 移除列       | 右侧选中后点"< 移除"             |
| 调整顺序      | 左键拖拽，或点"置顶"/"置底"         |
| 重命名列      | 双击右侧"新列名"列直接编辑           |
| 快速排序      | 右键某列 → "按此列升序/降序"        |
| 选中倒序      | Ctrl 多选几行 → 右键 → "选中项倒序" |
| 保存并复制为新任务 | 点击"保存并复制为新任务"会复制输出任务副本   |

#### 2.2.3 模式B：透视表

适用于"按部门汇总数量"、"按月份统计金额"等聚合场景。

| 字段 | 说明 | 示例 |
|------|------|------|
| 行标签 (Group By) | 按什么分组 | `部门,计划员` |
| 列标签 (Pivot Col) | 可选，把字段值变成列名 | `年份` |
| 列名:聚合函数 | 对哪个字段做什么运算 | `数量:sum, 金额:avg` |
| 重命名映射 | 可选，修改结果列名 | `可用库存=库存数量` |

**支持的聚合函数：** `sum`（求和）、`avg`/`mean`（平均）、`count`（计数）、`nunique`（去重计数）、`min`、`max`、`median`

**重命名映射的特殊处理：** 当透视表有列标签时，一个值列会被拆分成多列（如 `可用库存_2601`、`可用库存_2602`）。此时填写 `可用库存=库存数量`，程序会**批量替换所有匹配列**，结果变为 `库存数量_2601`、`库存数量_2602`。

### 2.3 Step 3：方案管理与启动

#### 2.3.1 保存方案

配置完成后，点击底部"保存"按钮，输入方案名称（如"月度库存报表"）。方案会保存当前所有配置：主表路径、匹配规则、输出任务、Schema 设置等。

> 如果输入的方案名称已存在，程序会提示"是否覆盖"。

#### 2.3.2 加载方案

从下拉框选择已保存的方案，自动恢复所有配置。

#### 2.3.3 导出方案（分享给同事）

选中一个方案 → 点击"导出"→ 保存为 `.json` 文件。把这个文件发给同事即可。

#### 2.3.4 导入方案（接收同事分享的方案）

点击"导入"→ 选择 `.json` 方案文件 → 自动加载。如果方案名称重复，可以选择覆盖或重命名。

> **分享方案的注意事项：** 方案中保存的是文件路径。同事导入后需要确认主表和目标表的文件路径在他的电脑上实际能访问到的路径（建议使用共享网络路径）。

#### 2.3.5 启动任务

1. 选择输出文件路径（右侧"选择"按钮）
2. 点击"启动任务"
3. 等待进度条完成
4. 完成后自动打开结果文件所在文件夹

#### 2.3.6 停止任务

运行中点击"停止"按钮（运行时变为红色），程序会在当前步骤完成后安全退出。已完成的结果不会丢失。

## 第三章 核心功能详解

### 3.1 多表关联匹配

这是 DataLink Pro 最核心的功能，等价于 Excel 的 VLOOKUP，但能力更强：

| 对比项 | VLOOKUP | DataLink Pro |
|--------|---------|-------------|
| 关联列数 | 只能 1 列 | 支持多列联合关联 |
| 取回方向 | 只能向右取 | 任意列任意方向 |
| 多表匹配 | 每个表写一遍公式 | 添加多条规则即可 |
| 重名处理 | 手动改列名 | 自动加前缀 |
| 性能 | 百万行卡死 | 百万行秒级 |

### 3.2 文件夹批量合并

适用场景：每个区域/门店/供应商交一份格式相同的 Excel，你需要先合并再处理。

工作原理：

1. 扫描文件夹内所有 xlsx/xls/csv/parquet 文件，自动跳过临时文件（~$ 开头）
2. 用最新文件作为列结构模板
3. 逐个文件校验列名——匹配的合并，不匹配的跳过并记录日志
4. 合并结果自动缓存，后续调参秒级加载

### 3.3 主表结构配置器

当主表列太多（几百列）但你只需要其中一部分时，用"结构"按钮打开配置器。

左右双栏设计：左侧是所有源列（可搜索），右侧是最终保留列（可排序）。右侧列的顺序就是虚拟列标 [A]/[B]/... 的顺序。

好处：

- 减少内存占用（只读取需要的列）
- 加快处理速度
- 理清数据结构

### 3.4 Parquet 缓存加速

V50 新增功能，解决大数据量读取慢的问题。

**什么是 Parquet？** 一种列式存储格式，读取速度比 Excel 快 10-50 倍，内存占用更低。

**使用方法：**

1. 点击主表旁边的"多缓存/单缓存"按钮
2. 选择包含 Excel/CSV 文件的文件夹/单个Excel/CSV 文件
3. 等待转换完成（每个文件转一次就够了）
4. 之后用文件夹模式加载同一文件夹，程序自动优先读取 Parquet 文件，或者直接选取单个已转Parquet的文件

**读取性能对比：**

| 数据规模 | Excel 读取 | Parquet 读取 |
|----------|-----------|-------------|
| 10万行 | 5-10秒 | <1秒 |
| 50万行 | 30-60秒 | 1-2秒 |
| 100万行 | 2-5分钟 | 2-3秒 |
| 500万行 | 通常失败 | 5-10秒 |

> 转换过程中原始 Excel 文件不会被删除，可以随时回退。

### 3.5 计算字段

在任务编辑窗口的"计算字段"输入框中填写公式。基本格式：

```
新列名 = 表达式
```

其中引用原表字段必须用 `df['列名']` 格式。多个公式用英文逗号分隔。

**常用公式速查：**

| 场景 | 公式 |
|------|------|
| 加法 | `总分 = df['语文'] + df['数学']` |
| 条件判断 | `状态 = np.where(df['库存'] < 10, '缺货', '充足')` |
| 文本拼接 | `编码 = df['区域'].astype(str) + '-' + df['门店号'].astype(str)` |
| 日期提取 | `月份 = pd.to_datetime(df['日期']).dt.month` |
| 空值填充 | `备注 = df['备注'].fillna('无')` |
| 新增空列 | `备注` |
| 固定值列 | `状态 = '正常'` |

完整 50 个公式示例请参见附录A，或点击程序内的"公式参考"按钮。

**辅助工具：**

- "插入列名"按钮：从可用数据源中选择列名，自动插入 `df['列名']` 格式
- "语法检查"按钮：写完公式后点击，检查语法是否正确

### 3.6 可视化筛选构建器

V50 新增功能。点击筛选条件旁的"构建"按钮打开。

**支持的运算符：**

| 运算符 | 说明 | 值输入方式 |
|--------|------|-----------|
| 等于 / 不等于 | 精确匹配 | 单值输入 |
| 大于 / 大于等于 / 小于 / 小于等于 | 数值比较 | 单值输入 |
| 包含文本 / 不包含文本 | 模糊匹配 | 单值输入 |
| 以...开头 / 以...结尾 | 前缀/后缀匹配 | 单值输入 |
| 在列表中 (in) / 不在列表中 (not in) | 多值匹配 | 多行文本框（支持从 Excel 粘贴） |
| 区间范围 (between) | 范围筛选 | 最小值 + 最大值 |
| 为空 / 不为空 | 空值判断 | 无需输入值 |

**使用方法：**

1. 下拉选择列名
2. 选择运算符（输入区域会自动切换）
3. 填写值（in list 模式支持每行一个值，或从 Excel 直接 Ctrl+V 粘贴）
4. 点击"添加条件"
5. 可添加多个条件，选择 AND（全部满足）或 OR（满足任一）
6. 底部显示最终表达式预览，支持手动微调
7. 点击"确认"，表达式自动填入筛选条件框

### 3.7 透视表

详见 2.2.3 节。补充说明：

- 透视表支持对结果进行排序和列重命名
- 当有列标签时，重命名映射会**批量替换**所有相关列
- 程序会先尝试 Polars 引擎（更快），失败自动回退到 Pandas

### 3.8 排序功能

两种排序方式：

**方式一：排序对话框**

点击"设置排序"按钮，弹出多重排序配置窗口。支持添加多个排序字段，越靠前优先级越高。双击可切换升序/降序。

**方式二：右键快速排序（V50新增）**

在明细表右侧输出列列表中，右键任意一列，弹出菜单：

- 按此列升序
- 按此列降序
- 选中项倒序（多选后批量倒序）
- 清除所有排序

### 3.9 紧急停止

V50 新增功能。运行任务时，"停止"按钮变为红色。点击后：

- 程序在当前步骤完成后安全退出
- 已完成的任务结果不会丢失
- 结果文件中会生成"程序执行报告"Sheet，记录中断原因

### 3.10 方案导出与导入

V50 新增功能。解决"同事之间共享配置方案"的需求。

**导出：** 选中方案 → 点击"导出"→ 保存为 `.json` 文件 → 发给同事

**导入：** 点击"导入"→ 选择 `.json` 文件 → 自动加载方案

> 注意：方案中保存的是文件路径。同事导入后需要确认路径在他的电脑上也能访问。建议使用共享网络路径。

### 3.11 运行日志与执行报告

**日志区：** 程序底部实时显示每一步的执行状态。绿色=成功，红色=错误。右上角"清空"按钮可清理历史日志。

**执行报告：** 每次运行完成后，结果 Excel 文件中会自动生成"程序执行报告"Sheet，记录每个阶段的时间、状态、详情。出了问题可以追溯。

## 第四章 常见问题与故障排除

| 问题 | 原因与解决方法 |
|------|---------------|
| 首次启动被杀毒软件拦截 | 因为是未签名的 exe。点击"更多信息"→"仍要运行"。或者在杀毒软件中添加信任。 |
| 点击启动后无反应 | 检查日志区是否有错误。最常见原因是主表或目标文件路径失效（文件被移动/删除）。 |
| 结果文件写入失败 | 输出文件正在被 Excel 打开。关闭 Excel 后重试。 |
| 匹配结果全是空值 | 检查主表关联列和目标关联列的值是否真的能对应上。常见原因：前导零被删除、空格差异、大小写不同。 |
| 内存不足 (OOM) | 用"结构"按钮只保留必需列。拆分任务或升级内存。确保使用 64 位系统。 |
| 缓存数据与实际不符 | 极少见。删除程序目录下的 `DataLink_Cache_Parquet` 文件夹即可。 |
| 计算字段报错 | 点击"语法检查"按钮验证。常见错误：列名拼写错误、缺少 `df['列名']` 格式、使用了中文括号/引号。 |
| 筛选条件报错 | 使用"构建"按钮生成表达式，避免手写语法错误。 |
| 文件夹模式提示"Sheet索引超出范围" | 文件夹内某个文件格式异常。程序会自动跳过，查看日志确认哪个文件有问题。 |
| 大文件读取太慢 | 使用"转缓存"按钮将 Excel 转为 Parquet 格式，之后读取速度提升 10-50 倍。 |

**求助方式：** 截取日志区的错误信息 + 你的操作步骤，发给开发。程序目录下的 `datalink_v50_error.log` 文件也包含详细错误日志。
